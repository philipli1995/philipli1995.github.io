如何监控jenkins

监控jenkins是为了了解jenkins的运行状态、性能、资源使用情况等，以及及时发现和处理可能出现的问题或者故障。有多种方法可以监控jenkins，例如：

使用 Monitoring 插件

Monitoring 插件（monitoring）使用 JavaMelody，对 Jenkins 进行监控。12 安装好插件后，可以在 Manage Jenkins 菜单下找到 Monitoring of Jenkins master 菜单项，单击菜单项进入后，显示 Monitoring 仪表盘，可以查看以下信息：

内存、CPU、HTTP 响应时间、当前请求数等
系统负载、线程数、打开的文件描述符等
已安装的插件、JVM 参数、环境变量等
构建历史、构建时长、构建错误等
该插件提供了丰富的监控维度，但是没有告警功能。

使用 Prometheus 监控

Prometheus 是一款开源的监控、告警系统。3 Prometheus 实现了与 Zabbix 或者 Open-Falcon 类似的功能，但更强大。Prometheus 采用的是 pull 模式收集指标数据的，即 Prometheus 的服务器端主动从客户端拉取指标数据。这个客户端被称为 exporter。

我们需要在 Jenkins 上安装 Prometheus 插件（prometheus），目的就是为了暴露一个接口（exporter），这样 Prometheus 就可以拉取到指标数据了。3

安装好插件后，需要进行配置，“系统管理”——>“系统设置”页面。通过此配置，我们可以选择暴露接口的 URL，以及暴露哪些指标数据。

然后，在 Prometheus 的配置文件中添加对 Jenkins 的数据采集：

metrics_path 是 Jenkins 暴露给 Prometheus 的路径。 static_configs 数组的值是 Jenkins 的 ip 地址：端口。

配置完成后重新加载配置文件，并且在 Prometheus 中检查指标是否被采集。

最后，在 Grafana 中配置图表进行展示，导入模板，这里导入的模板 id 是 9964 ，就可以看到 Jenkins 的各种指标数据了，并且可以设置告警规则和通知方式。

使用 Zabbix 监控

Zabbix 是一款开源的分布式监控系统。1 Zabbix 采用的是 push 模式收集指标数据的，即 Zabbix 的客户端主动向服务器端发送指标数据。

要使用 Zabbix 监控 Jenkins ，需要在 Jenkins 上安装 Metrics 插件（metrics），使 Jenkins 暴露 metrics api 。1

然后，在 Zabbix server 和 agent 上进行相应的配置以实现监控和告警。具体步骤如下：

  • 在 Zabbix server 上创建一个名为 jenkins-master 的主机，并添加一个名为 jenkins-master-template 的模板。
  • 在模板中创建一个名为 jenkins-master-application 的应用程序，并添加以下项目：
    • jenkins.master.healthcheck：检查 Jenkins 主节点健康状况V
    • jenkins.master.version：获取 Jenkins 版本号
    • jenkins.master.plugins.count：获取已安装插件数量
    • jenkins.master.plugins.updates.count：获取可更新插件数量
  • 在每个项目中设置类型为 Zabbix agent，键值为 jenkins.metrics [<参数>]，例如：jenkins.metrics [gauges.jenkins.node.count.value.value]，表示获取 Jenkins 节点个数。
  • 在每个项目中设置数据类型、更新间隔、历史记录保留时间等参数。
  • 在模板中创建触发器（trigger），设置告警条件和消息通知方式。
  • 在 Zabbix agent 所在机器上放置一个 Python 脚本（jenkins.metrics.py），用于从 Jenkins 的 metrics api 获取指标数据，并解析成 Zabbix 可识别的格式。
  • 在 Zabbix agent 的配置文件中添加用户自定义参数（UserParameter），将键值与脚本对应起来，例如：UserParameter=jenkins.metrics [*], python /usr/lib/zabbix/externalscripts/jenkins.metrics.py
